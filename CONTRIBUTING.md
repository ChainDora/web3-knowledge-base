# 贡献指南 / Contributing

非常欢迎你一起维护这个知识库。这是一个公共教育项目，不是商业付费课，所以你的贡献会真正帮助到很多新人、开发者和研究员。

## 我可以怎么贡献？

### 1. 文档改进
- 补充解释 / 画流程图 / 翻译英文资料
- 修正错误或过时描述
- 增加风险提示（比如：钱包安全、跨链桥安全、RWA合规）

流程：
1. Fork 本仓库
2. 在 `docs/` 或 `tools/` 下面改动
3. 提交 PR（Pull Request）
4. 在 PR 里简单写明你改了什么、为什么改

### 2. 示例代码
- 最小可运行合约（ERC20、ERC721A、ERC3643等）
- 演示 AA / EIP-4337 的 demo
- 简单 AMM / 闪电贷示例

请把这些放进 `examples/` 目录，并附上 README 解释如何运行。

### 3. 安全 & 审计经验
- 常见漏洞案例（重入、溢出、签名复用、预言机操纵等）
- 安全文档、自查 checklist
- DAO 治理攻击、跨链桥攻击的复盘

可以放在：
- `docs/04_smart_contracts/` 的安全小节
- 或 `tools/` 的 checklist

### 4. 补充术语
如果你想补充/修正术语表，请直接改 `glossary/glossary.md`。

---

## 写作风格建议
- 面向“第一次接触这个词的人”
- 多解释“为什么重要”，而不只是“是什么”
- 尽量举真实项目案例（Uniswap / MakerDAO / Arbitrum / Chainlink / etc.）
- 标注标准号（比如：EIP-1559、EIP-4337、ERC-3643）
- 不发投机/拉盘类内容
